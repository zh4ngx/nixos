## Directory Responsibilities

**`hosts/`**

Per-host entry points. Each host imports generated hardware config, shared
modules, and any host-only hardware modules.

**`modules/nixos/`**

System-level configuration shared across hosts: Nix settings, sops-nix, desktop
services, networking, sudo, auto-upgrade, caches, system packages, and shared
hardware modules.

**`modules/home-manager/`**

User-level configuration for Andy: packages, dotfiles, fish functions, user
services, zellij layouts, agent launchers, XDG files, and per-application Home
Manager modules.

**`packages/`**

Local package derivations. Use this when a needed tool is not already consumed
cleanly from nixpkgs or a flake input. There are no active local package
derivations at the moment.

**`secrets/`**

Encrypted secret material and secret templates. Do not put plaintext secrets in
this directory or in the Nix store.

**`agents/`**

Shared agent instructions and agent-specific reference docs. `AGENTS.md` is the
canonical operational policy file for agents working in this repo.

**`docs/`**

Longer-form human documentation and historical notes. Some docs here may be
older than the root orientation files.

**`files/`**

Static files deployed by Home Manager.

## Important Entry Points

- `flake.nix`: top-level flake inputs and host orchestration
- `hosts/<hostname>/default.nix`: host-specific NixOS entry point
- `modules/nixos/default.nix`: shared system config
- `modules/home-manager/default.nix`: shared user config and most agent wiring
- `agents/AGENTS.md`: canonical agent rules
- `secrets/secrets.yaml`: encrypted secret source

## Naming Conventions

- `default.nix`: module or package entry point
- `hardware-configuration.nix`: generated per-host hardware config
- `modules/home-manager/<app>.nix`: app-specific Home Manager module
- `modules/nixos/hardware/<device>.nix`: shared hardware support module
- `packages/<name>/default.nix`: local package derivation

## Where To Add Things

**New host**

Add `hosts/<hostname>/`, add generated `hardware-configuration.nix`, and include
the hostname in `flake.nix`.

**New shared NixOS behavior**

Add it to `modules/nixos/default.nix` or a focused module imported from a host
or shared module.

**New Home Manager behavior**

Add it to `modules/home-manager/default.nix` when it is central user wiring, or
create `modules/home-manager/<name>.nix` when it is large enough to stand alone.

**New local package**

Add `packages/<name>/default.nix` and consume it with `pkgs.callPackage` from the
module that needs it. Expose it as a flake package only if external consumers
need that interface.

**New flake input package**

Add the input in `flake.nix`, follow `nixpkgs` when appropriate, then consume
`inputs.<name>.packages.${pkgs.stdenv.hostPlatform.system}.<package>` or an
upstream module.

**New secret**

Edit `secrets/secrets.yaml` with sops, declare the secret in
`modules/nixos/default.nix`, and consume it through `/run/secrets/...` or a
sops-rendered template.

**New agent route or launcher**

Prefer the canonical abstraction first:

- MetaStack route: `programs.metastack.routingConfig`
- Agent launcher or zellij layout: `modules/home-manager/default.nix`
- Agent policy: `agents/AGENTS.md`
- Cross-project writes: dispatch to the owning project agent

**New MCP server**

Use `programs.mcp.servers` for shared non-secret MCP servers. Use a
secret-rendered config or tool-specific config when the server needs secrets.
